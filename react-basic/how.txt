패키지 관리자 명령문 이해하기
-npm npx yarn 기본설치x 들은 모두 npde 프로젝트에서 설치되는 패키지 관리 툴

npm
자바스크립트 런타임 환경 Node.js의 기본 패키지 관리자
자바스크립트 환경에서 프로젝트를 시작할때 노드와 함께 설치하라는 Getting started가 많고 사용빈도도 높다.

npx
npx (excuted npm binaries) npm에 속해 있는 패키지 실행도구이다
npx는 패키지 실행도구라는 점이 npm이 과 차이점
다른 사람의 깃에있는 프로젝트를 다운받아서 package.json의 script중 하나를 실행하고 싶을때 패키지가 설치되어야 한다
이럴때 npx와 스크립트를 적으로면 패키지 설치를 하지 않고실행이 가능하다

yarn
안전하고 안적적으로 재현이 가능한 프로젝트다 yarn은 npm,npx와 결을 달리하는 패키지 관리자이다

속도나 안정성에서 npm과 비슷하거나 더 나은 효과를 가져온다.

차이점은 버전을 어디에서나 같게 만들어서 버그를 줄여주고 보안성이 높다는 점이다.
yarn은 기본적으로 설치되어 있지않아서 yarn 패키지 관리자를 설치해야한다.

--- 명령어 ---
npm install 패키지면 <태그|버전|버전범위> 특정패키지 설치를 진행한다.
npm install은 package.json에 있는 패키지들을 설치
